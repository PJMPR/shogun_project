\section{Cel dydaktyczny}

Celem przedmiotu jest zapoznanie studentów z metodologią Infrastructure as Code (IaC) oraz narzędziami i platformami umożliwiającymi automatyczne tworzenie, konfigurowanie i zarządzanie infrastrukturą chmurową na skalę produkcyjną. Student nabywa praktyczne umiejętności pracy z Terraform (provisionowanie zasobów), Ansible (zarządzanie konfiguracją i automatyzacja), Amazon Web Services (usługi chmury publicznej) oraz OpenStack i jego lokalnym emulatorem LocalStack (chmura prywatna i środowiska testowe). Przedmiot kształci kompetencje inżyniera DevOps/Cloud, który potrafi projektować powtarzalne, wersjonowane i audytowalne środowiska infrastrukturalne.

\subsection*{\textit{Wersja w języku angielskim (English version)}}

\textit{The aim of the course is to familiarize students with the Infrastructure as Code (IaC) methodology and the tools and platforms enabling the automatic creation, configuration, and management of cloud infrastructure at production scale. Students acquire practical skills in working with Terraform (resource provisioning), Ansible (configuration management and automation), Amazon Web Services (public cloud services), and OpenStack together with its local emulator LocalStack (private cloud and testing environments). The course develops the competencies of a DevOps/Cloud engineer who can design repeatable, versioned, and auditable infrastructure environments.}

\section{Przedmiotowe efekty uczenia się}

\subsection*{Wiedza}
\begin{itemize}
  \item Student zna i rozumie koncepcję Infrastructure as Code (IaC): zasady deklaratywnego i imperatywnego podejścia do definiowania infrastruktury, korzyści z wersjonowania i powtarzalności środowisk.
  \item Student zna i rozumie architekturę i model działania Terraform: providers, resources, data sources, state (lokalny i zdalny), moduły, workspace oraz cykl plan–apply–destroy.
  \item Student zna i rozumie zasady zarządzania konfiguracją z użyciem Ansible: architektura (control node, managed nodes, inventory), playbooks, role, zmienne, szablony Jinja2 i idempotentność operacji.
  \item Student zna i rozumie model usług chmurowych (IaaS, PaaS, SaaS) oraz globalną infrastrukturę AWS: regiony, strefy dostępności, VPC, subnety, grupy bezpieczeństwa.
  \item Student zna i rozumie kluczowe usługi AWS wykorzystywane w automatyzacji infrastruktury: EC2, S3, RDS, IAM, ELB, Auto Scaling, Route 53, CloudWatch, ECR/ECS.
  \item Student zna i rozumie architekturę OpenStack: komponenty (Nova, Neutron, Cinder, Glance, Keystone, Swift, Heat) i ich odpowiedniki w usługach AWS.
  \item Student zna i rozumie przeznaczenie i możliwości LocalStack jako lokalnego emulatora usług AWS, umożliwiającego testowanie infrastruktury bez dostępu do chmury publicznej i bez kosztów.
  \item Student zna i rozumie zasady bezpiecznego zarządzania dostępem w chmurze: IAM (użytkownicy, role, polityki, least privilege), zarządzanie kluczami i sekretami (AWS Secrets Manager, HashiCorp Vault), szyfrowanie zasobów.
  \item Student zna i rozumie strategie zarządzania stanem infrastruktury: remote state (S3 + DynamoDB), state locking, import istniejących zasobów, techniki refaktoryzacji konfiguracji Terraform.
  \item Student zna i rozumie podstawy monitorowania i obserwowalności infrastruktury chmurowej: metryki, logi, alerty (CloudWatch, Prometheus, Grafana) oraz koncepcję SLI/SLO/SLA.
\end{itemize}

\subsection*{Umiejętności}
\begin{itemize}
  \item Student potrafi zainstalować i skonfigurować środowisko pracy z Terraform, Ansible i LocalStack oraz nawiązać połączenie z klastrem OpenStack.
  \item Student potrafi napisać konfigurację Terraform definiującą sieć (VPC, subnety, routing, security groups), maszyny wirtualne oraz zarządzane usługi bazodanowe w AWS lub ich odpowiedniki w LocalStack.
  \item Student potrafi stosować moduły Terraform do organizowania i wielokrotnego użycia konfiguracji infrastruktury, zarządzać stanem zdalnym w S3 z blokadą w DynamoDB.
  \item Student potrafi napisać playbook Ansible instalujący i konfigurujący serwer aplikacyjny (np. Nginx + Node.js/Python) na zdalnych hostach, stosując role, zmienne i szablony Jinja2.
  \item Student potrafi zintegrować Terraform z Ansible w celu automatycznego provisionowania maszyn i ich konfigurowania w jednym potoku wdrożeniowym.
  \item Student potrafi uruchomić LocalStack i za pomocą AWS CLI oraz Terraform przetestować lokalnie infrastrukturę obejmującą S3, SQS, Lambda, DynamoDB i EC2.
  \item Student potrafi tworzyć i zarządzać zasobami OpenStack (instancje, sieci, woluminy, floating IP, security groups) z poziomu CLI (openstack client) oraz za pomocą szablonów Heat.
  \item Student potrafi skonfigurować zarządzanie dostępem IAM w AWS: tworzyć użytkowników, role i polityki uprawnień zgodnie z zasadą najmniejszych uprawnień.
  \item Student potrafi skonfigurować monitoring infrastruktury: tworzyć alarmy CloudWatch, eksportować metryki do Prometheus i wizualizować je w Grafana.
  \item Student potrafi zbudować potok CI/CD (GitHub Actions) automatycznie weryfikujący (terraform plan, ansible-lint) i wdrażający zmiany infrastruktury po zatwierdzeniu pull requesta.
\end{itemize}

\subsection*{Kompetencje społeczne}
\begin{itemize}
  \item Student jest gotów do traktowania infrastruktury jako kodu – przechowywania konfiguracji w systemie kontroli wersji, przeprowadzania code review zmian infrastrukturalnych i stosowania standardów branżowych (formatowanie, dokumentacja modułów).
  \item Student jest gotów do świadomego zarządzania kosztami chmury publicznej: rozumie model rozliczeń pay-as-you-go, potrafi szacować koszty planowanych zasobów i stosować mechanizmy ograniczania wydatków (budgets, reserved instances, auto scaling).
  \item Student jest gotów do samodzielnego pogłębiania wiedzy z zakresu dynamicznie rozwijającego się ekosystemu chmurowego, śledzenia dokumentacji dostawców (AWS, OpenStack, HashiCorp) i certyfikowania swoich kompetencji (AWS Cloud Practitioner, Terraform Associate).
\end{itemize}
