\section{Cel dydaktyczny}

Celem przedmiotu jest pogłębienie wiedzy studentów z zakresu współczesnych silników gier na przykładzie Unity 6, z uwzględnieniem porównania z Unreal Engine oraz Godot w kontekście doboru technologii do skali i typu projektu. Studenci poznają zaawansowane systemy silnika Unity, takie jak render pipelines (URP/HDRP/Built-In) i różnice między nimi, architekturę ECS/DOTS i jej wpływ na wydajność, sposoby zarządzania zasobami metodami Asset Bundle, Addresables i Streaming Assets, New Input System i Unity Localization Package. Studenci poznają także dobre praktyki i architektury które wesprą ich przy integracji Unity z zewnętrznymi platformami. 

\subsection*{\textit{Wersja w języku angielskim (English version)}}

\textit{The aim of the course is to deepen students’ knowledge of contemporary game engines using Unity 6 as the main example, with comparisons to Unreal Engine and Godot in the context of selecting technology appropriate to the scale and type of project. Students learn advanced Unity engine systems such as render pipelines (URP/HDRP/Built-In) and the differences between them, the ECS/DOTS architecture and its impact on performance, asset management methods such as Asset Bundles, Addressables, and Streaming Assets, the New Input System, and the Unity Localization Package. Students also learn good practices and architectural approaches that support Unity integration with external platforms.}

\section{Treści programowe}

\begin{enumerate}
  \item Przegląd i porównanie silników gier
  \item Konfiguracja i zastosowanie URP/HDRP, dobór pipeline'u do projektu
  \item Architektura ECS/DOTS
  \item Zarządzanie zasobami - Asset Bundles, Steaming Assets i Addressables
  \item New Input System
  \item Lokalizacja projektu - Integracja pakietu lokalizacji Unity
  \item Architektura projektu, modularność, wzorce projektowe stosowane w produkcji gier
  \item Optymalizacja i przygotowanie buildów produkcyjnych
  \item Proces wydawniczy, przygotowanie projektu do publikacji
  \item Projekt zaliczeniowy - implementacja i prezentacja
\end{enumerate}

\section{Kryteria oceny}

\begin{itemize}
  \item Projekt gry lub aplikacji w dowolnym przedstawionym na zajęciach silniku, realizowany w zespołach 2-3 osobowych.
  \item Prezentacja projektu oraz dokumentacji projektowej na zajęciach. Ocena obejmuje poprawność działania aplikacji, optymalne zarządzanie zasobami urządzenia docelowego, dokumentacja projektowa i stopień przygotowania wersji aplikacji do wydania
  \item Warunkiem zaliczenia jest oddanie kompletnego projektu i ich pozytywna ocena.
\end{itemize}
